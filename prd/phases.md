### Phase 1 — Roadmap only
Use /goal to create sprints, risks, acceptance criteria, API plan, UI plan, testing plan, and storage model. No code.

### Phase 2 — Architecture only
Use /task to design backend, frontend, file store, skill lifecycle, LiteLLM routing, privacy modes, audit logging, and rollback. No code.

### Phase 3 — Sprint implementation
Implement one sprint at a time only.

### Phase 4 — Review after every sprint
Run tests, compare against PRD, check safety rules, check no SQL/login/cloud database was added.

### Phase 5 — Final MVP hardening
Test local-only mode, document extraction, skill approval, generation, rewrite, audit log, rollback, and packaging.

### Phase 6
Continue until all roadmap sprints are complete. Do not assume exactly six sprints.
